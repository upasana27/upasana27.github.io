Who is Helping Whom?
Analyzing Inter-dependencies to Evaluate Cooperation in Human–AI Teaming

Upasana Biswas, Vardhan Palod, Siddhant Bhambri, Subbarao Kambhampati

School of Computing and AI, Arizona State University

AAAI 2026 Also presented at CoCoMARL Workshop, RLC 2025
arXiv OpenReview Code

TL;DR

State-of-the-art zero-shot cooperation agents achieve high task rewards but exhibit remarkably low cooperation. We propose constructive interdependence—a formal metric grounded in STRIPS—that reveals this gap. A 36-participant user study confirms that task reward alone is an unreliable indicator of teamwork.

Motivation

In cooperative environments like Overcooked, an AI agent can achieve high rewards without meaningfully helping its human partner. The human may compensate for an idle or non-adaptive agent, resulting in high team scores that conceal deficiencies in the agent’s cooperative abilities.

This is especially problematic in Non-Required Cooperation (Non-RC) settings, where one player can complete the task alone while the other remains idle. Both strategies can yield similar task rewards despite vastly different levels of cooperation. We need metrics that evaluate how agents support and depend on one another—not just whether the task succeeds.

Approach: Constructive Interdependence

We formalize cooperation by mapping a two-player Markov game to a symbolic STRIPS representation. Each agent’s action is expressed as ⟨preconditions, add-effects, delete-effects⟩, letting us systematically track when one agent’s action effects satisfy another’s preconditions.

We define constructive interdependence as a pair of actions (ai, aj) where:

  • Agent j’s action effect satisfies agent i’s precondition (interdependence)
  • The interaction contributes to task completion (goal-reaching)
  • The object doesn’t return to the giver in the same state (non-looping)

Example: Agent j places an onion on the counter (add-effect: onion-on-counter) → Agent i picks it up (precondition: onion-on-counter) and puts it in the pot. This is a constructive interdependence linked to the onion object.

The metric is domain-agnostic. We provide a fully modular software package with a Mapping Module (raw trajectories → symbolic STRIPS traces) and an Analysis Module (detects and categorizes interdependencies). We also demonstrate it on a heterogeneous Search and Rescue domain.

Experimental Setup

We evaluate four state-of-the-art ZSC agents—FCP, MEP, HSP, and COLE—in the Overcooked domain across two layouts:

  • Forced Coordination (RC) — cooperation is structurally required
  • Counter Circuit (Non-RC) — agents can complete the task independently

Each agent is paired with: (1) a scripted cooperative partner, (2) a copy of itself (self-play), (3) learned human models (Hproxy), and (4) real human participants in a user study with 36 participants (ages 18–31, 24 male, 12 female, 64% with no prior Overcooked experience).

Key Results

High reward ≠ high cooperation. MEP achieves the highest task reward (43.33) paired with a cooperative partner, but its constructive interdependence (Intcons = 0.83) is far below the cooperative upper bound (6.59). HSP and FCP show zero constructive interdependence with 100% rejection rates of cooperation attempts.

Finding 1: ZSC agents reject cooperation

When paired with a scripted partner that consistently initiates coordination, ZSC agents reject 76–100% of these attempts. Despite the partner’s consistent cooperative behavior, agents achieve rewards by acting independently.

Finding 2: RC vs. Non-RC reveals the gap

  • In RC layouts: Task reward and Intcons correlate strongly (COLE: reward=200, Intcons=30.43) because cooperation is structurally forced.
  • In Non-RC layouts: The correlation breaks. COLE and HSP both achieve reward=120 in self-play, but Intcons differs by 5× (10.23 vs. 1.82).

Finding 3: Humans try, agents don’t reciprocate

With real human partners, constructive interdependencies remain consistently low in Non-RC settings. COLE achieves reward=76.21 but Intcons=1.89. Humans frequently attempt to initiate cooperation (60–67% of the time), but ZSC agents ignore most of these signals.

Finding 4: The metric can improve training

Incorporating interdependence as an additional reward signal during training leads SOTA agents to consistently increase constructive interdependencies without harming task performance, suggesting the metric can be used to promote genuine cooperation.
